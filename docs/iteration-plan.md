# 迭代计划

## 总体原则

- 先文档和规则，后工程初始化
- 先 P0 闭环，后增强功能
- 先做能演示的成果，后做技术亮点
- 功能建设和内容建设同步推进

## 阶段 0：项目基线

目标：

- 建立 GitHub 仓库
- 明确需求、架构、迭代顺序
- 固化 Codex 协作规则

产出：

- `README.md`
- `AGENTS.md`
- `docs/requirements.md`
- `docs/iteration-plan.md`
- `docs/codex-prompts.md`
- `docs/content-plan.md`

完成标准：

- 仓库已有基础文档
- 项目默认规则已明确

## 阶段 1：工程初始化

目标：

- 初始化后端工程
- 初始化前端工程
- 建立数据库初始化脚本

产出：

- `blog-server`
- `blog-web`
- `sql/init.sql`

完成标准：

- 前后端可启动
- 数据库表可初始化

## 阶段 2：后台登录

目标：

- 完成登录、Token 校验、退出登录

完成标准：

- 登录成功后可进入后台
- 未登录时无法访问受保护页面
- 退出后登录态失效

## 阶段 3：后台内容管理

目标：

- 完成分类、标签、文章管理

完成标准：

- 可新增、编辑、删除分类和标签
- 可新增、编辑、删除文章
- 支持草稿、发布、下线
- 支持文章类型管理和专题标识录入

## 阶段 4：前台展示

目标：

- 完成首页、文章列表、文章详情、分类、标签、搜索

完成标准：

- 已发布文章可正常浏览
- Markdown 渲染正常
- 搜索和筛选可用
- 首页具备学习路线、学习总结、面试复盘、项目实战入口

## 阶段 4.5：首批内容建设

目标：

- 在系统具备基础展示能力后，补齐首批内容

完成标准：

- 至少形成一批基础文章选题
- 至少完成学习总结、项目实战、面试复盘 3 类内容的初始沉淀
- 博客上线时不是空壳

## 阶段 5：增强功能

目标：

- 完成系统设置、统计、上传、阅读量、热门文章

## 阶段 6：技术亮点

目标：

- 完成 Redis、评论/点赞、日志、MQ 或微服务扩展中的一部分

## 每日执行节奏

每天建议按下面流程推进：

1. 明确当天目标
2. 按项目默认规范开发
3. 完成本地自测
4. 更新日报
5. 提交并推送 GitHub
