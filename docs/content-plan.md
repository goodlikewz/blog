# 内容体系与选题规划

## 一、内容策略定位

这个博客不能只是一个可发布文章的系统，还必须是一个持续输出内容的学习平台。

内容建设要同时服务 3 个目标：

- 学习总结：把 Java 学习过程沉淀成体系化文章
- 面试准备：把高频难点和场景题沉淀成可复习内容
- 项目表达：把博客开发过程中的设计、踩坑、优化沉淀成项目实战文章

## 二、核心内容栏目

建议第一阶段优先建设以下栏目：

- Java 基础
- Spring Boot
- Spring Cloud
- Redis
- RocketMQ
- MySQL
- JVM
- 项目实战
- 面试复盘
- 踩坑记录
- 学习总结

## 三、文章类型设计

建议文章支持以下类型：

- 学习笔记
- 学习总结
- 源码解析
- 项目实战
- 面试复盘
- 场景题拆解
- 踩坑记录

作用：

- 让内容不只是按技术分类，还能按“写作目的”归类
- 更适合后续做面试专题和学习路线专题

## 四、首页内容结构建议

首页建议不只展示最新文章，还应体现学习路径和内容体系。

推荐模块：

- 顶部介绍区
- 学习路线导航
- 最新文章
- 学习总结
- 场景面试专题
- 项目实战专题
- 分类导航
- 标签云
- 关于我 / GitHub

## 五、面试场景内容策略

当前 Java 面试越来越强调场景题、系统设计题、优化题和排障题，因此博客应单独建设“场景面试”内容方向。

建议重点沉淀以下类型内容：

- 高并发场景
- 缓存一致性场景
- 库存扣减场景
- 消息重复消费场景
- 分布式锁场景
- 慢 SQL 排查场景
- 大数据导出优化场景
- 登录鉴权场景
- 接口幂等场景
- 项目故障排查场景

每篇场景类文章建议结构：

1. 问题背景
2. 真实业务场景
3. 常见错误做法
4. 可行方案对比
5. 推荐实现方案
6. 面试回答思路

## 六、学习总结内容策略

博客不能只有零散笔记，还需要定期输出“阶段总结型文章”。

建议固定产出：

- Java 基础阶段总结
- Spring Boot 学习总结
- Redis 学习总结
- RocketMQ 学习总结
- 博客项目阶段总结
- 每周 / 每月学习复盘

每篇学习总结建议结构：

1. 这阶段学了什么
2. 核心知识点
3. 容易混淆的问题
4. 实践中的理解
5. 后续待补内容

## 七、项目实战内容策略

建议把博客项目本身也作为内容来源，持续输出项目开发文章。

可写主题：

- 为什么选择做这个博客项目
- 博客项目如何做需求拆分
- 博客项目数据库如何设计
- 后台登录为什么采用 JWT
- 文章搜索第一阶段为什么先不用 Elasticsearch
- Redis 如何接入热门文章
- 博客系统如何逐步演进到微服务

## 八、第一批建议选题

建议第一批至少准备 12 到 20 篇内容，保证博客上线时不是空壳。

推荐起步选题：

- Java 集合常见面试问题总结
- synchronized 和 ReentrantLock 的区别
- Spring Boot 自动装配核心理解
- MyBatis-Plus 常用开发模式总结
- MySQL 索引失效的常见场景
- Redis 常见面试题整理
- RocketMQ 为什么适合异步解耦
- JWT 登录流程与实现思路
- 分布式锁常见方案对比
- 慢 SQL 排查实战总结
- 接口幂等实现方案总结
- 大批量导出优化思路
- 项目里为什么需要统一返回结构
- Java 博客项目需求分析
- 博客系统数据库设计说明
- 博客系统登录模块设计
- 博客系统文章模块设计
- 第一阶段开发总结

## 九、内容与功能联动建议

后续功能设计应服务内容表达，建议逐步支持：

- 文章类型筛选
- 学习路线聚合页
- 面试专题聚合页
- 项目实战聚合页
- 文章归档页
- 推荐阅读

## 十、当前阶段执行建议

在第一阶段开发期间，同步完成两条线：

- 功能线：先完成博客系统 P0 闭环
- 内容线：至少准备一批基础文章和专题结构

原则：

- 不让博客只有页面，没有内容
- 不让内容只有草稿，没有沉淀结构
