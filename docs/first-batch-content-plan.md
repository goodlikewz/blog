# 第一批内容建设计划

## 一、目标

本计划用于解决两个问题：

- 博客第一版上线时不能只有系统壳子，没有内容
- 内容建设要直接服务 Java 学习、场景面试和项目展示

建议第一批至少准备 20 篇文章，其中：

- 学习总结类 6 篇
- 场景面试类 6 篇
- 项目实战类 5 篇
- 踩坑记录 / 排障总结类 3 篇

原则：

- 先有基础内容，再逐步扩充深度
- 先覆盖高频面试与高频学习主题
- 文章要能复用到面试表达和项目复盘中

## 二、内容分组

### 2.1 学习总结类

定位：

- 用于沉淀阶段性学习成果
- 帮助建立知识体系，而不是零散笔记

### 2.2 场景面试类

定位：

- 面向当前 Java 面试中的高频场景题
- 训练“问题背景 - 分析过程 - 方案取舍 - 落地实现”的表达能力

### 2.3 项目实战类

定位：

- 用博客项目本身作为素材
- 形成可直接用于面试项目讲解的内容资产

### 2.4 踩坑记录类

定位：

- 记录真实问题、排查思路和解决过程
- 强化问题定位与故障分析能力

## 四、上线前最低内容标准

建议博客第一版上线前至少满足：

- 学习总结类文章不少于 3 篇
- 场景面试类文章不少于 3 篇
- 项目实战类文章不少于 2 篇
- 踩坑记录类文章不少于 1 篇

总计至少 9 篇。

这样可以避免：

- 首页空洞
- 分类页内容太少
- 面试专题页没有实质文章
- 项目实战页只有标题没有内容

## 六、写作节奏建议

建议系统开发和内容输出同步推进。

推荐节奏：

- 每完成一个核心功能模块，至少补 1 篇项目实战文章
- 每学习完一个技术阶段，至少补 1 篇学习总结文章
- 每周至少补 1 篇场景面试文章

推荐每周最少产出：

- 2 篇文章

推荐组合：

- 1 篇项目 / 学习总结
- 1 篇场景面试 / 踩坑记录
