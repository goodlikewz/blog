# AGENTS.md

## 项目定位

本项目是一个学习型技术博客系统。

它不是单纯的网站项目，而是同时承担以下目标：

- 构建一个可运行、可上线的个人博客
- 练习 Java 后端开发能力
- 沉淀技术学习内容
- 形成可用于面试展示的项目作品
- 建立持续输出学习总结与面试场景文章的内容体系

## 默认技术栈

- 后端：Spring Boot + MyBatis-Plus + MySQL + JWT
- 前端：Vue 3 + Vite + Element Plus
- 架构：前后端分离，RESTful API
- 第一阶段：优先单体应用，不做权限细分，不做微服务拆分

## 默认协作方式

- 用户通常只给目标，不重复描述技术栈
- 默认由 Codex 负责：代码实现、SQL 变更、配置补充、前后端联调、本地自测
- 完成编码后默认继续做验证，而不是停在“代码已生成”
- 如存在可运行条件，默认尝试启动前后端并进行基础流程验证
- 如遇到重大方案分叉，再向用户确认

## 优先级策略

- 第一优先级是做出 P0 可运行闭环
- 第二优先级是保证后端结构规范，适合 Java 学习和面试表达
- 第三优先级是支持长期写作与内容沉淀
- 第四优先级是逐步形成面向场景面试的问题库与总结库
- 不提前引入复杂权限、微服务、过度设计

## 工程原则

- 目录结构清晰，便于后续扩展
- 优先保证可运行和可验证
- 代码应体现 Java 项目分层与工程化能力
- 新增或修改数据表时，同步维护 SQL 文档
- 每个阶段都应形成可演示成果
- 功能设计要服务内容沉淀，避免只做空壳博客
- 内容体系要能支撑学习总结、项目实战、场景面试 3 类核心内容

## 语言规范

- 整个项目默认以中文为主
- 规则文档标题、章节名称、说明文字默认使用中文
- 项目文档、开发说明、日报、接口说明、提交总结默认使用中文
- 代码注释默认使用中文，优先写清业务意图和设计原因
- 后端类和方法默认使用 Javadoc 注释，格式参考：说明文本、空行、`@param`、`@return`
- 方法内部关键步骤默认使用中文 `//` 单行注释，注释写在对应代码上一行
- 方法内部注释风格遵循“先说明这一行/这一段做什么，再写代码”
- 简单 getter/setter、字段赋值、明显语句不强制逐行注释
- 注释重点说明“这个模块做什么”“当前步骤为什么这样处理”“异常或边界场景如何处理”
- 页面文案、后台管理文案、表单提示信息默认使用中文
- 数据库表注释和字段业务说明默认使用中文
- 非必须情况下，不主动引入英文注释或英文说明
- 代码中的技术标识符保持行业常见命名方式，如类名、方法名、变量名、接口路径仍可使用英文
- 若引用框架约定、第三方库配置项、标准技术术语，可保留英文原文

## Git 工作流

- 每次功能开发完成后，默认进行本地自测
- 每天开发结束后，更新 `docs/daily/YYYY-MM-DD.md`
- 每天至少进行一次 Git 提交
- 远程仓库已配置时，默认推送到 GitHub
- 提交信息优先使用 `feat` / `fix` / `docs` / `refactor` / `chore`

## 输出要求

每次任务完成后，默认输出：

- 完成了什么
- 关键改动点
- 如何运行或验证
- 自测结果
- 下一步建议
